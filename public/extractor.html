Order Validation Dashboard - Enterprise FINAL V3

This browser-based system runs without Python. Users can open the file on a PC, extract player details from the raw order CSV, then upload the cleaned CSV file together with the packing list to generate a final packing list export with mismatch checks and red-highlighted review rows.

Processing Console

Player Details Extractor

Upload the raw order CSV. The system uses an adaptive extraction engine with segment classification, confidence scoring, and business-rule suppression to extract Player Number, Player Initial, and Player Name from Product Variation Details. Initial values like NO are kept when they are explicitly written as the player initial, and name extraction can also infer a clean unlabeled player name when it appears right beside player-related segments. Stage 1 exports a clean CSV, trims extra spaces, and forces Player Number columns to open in Excel as text so values like 00 stay exactly as 00.

Validation Rule

Matching now uses Order ID + Material from the cleaned order file against BC Order # + Material in the packing list. Header names are trimmed automatically, so columns like "Material " also match correctly.

Name Rule

Product SKU is no longer the main key. The system uses the Material column from the cleaned order file, which fixes cases where Product SKU differs from the packing material.
